# First-Person Body System — Research & Implementation

**Last updated:** 2026-03-29

---

## 1. Problem Statement

**Symptom:** Camera is on the capsule (fixed offset). Motion Matching animates the skeleton with inertia/overshoot. On stops, the body slides forward — player sees inside the mesh.

**Hard constraints:**
1. Rigid camera on capsule — no spring arm, no bone attachment (causes jitter)
2. Full MM animation fidelity — no procedural blocking
3. Full body visible — legs/arms when looking down

**Solution:** Three-layer body system + root drift compensation (SpineLock).

---

## 2. Architecture

### 2.1 Three-Layer Pipeline

```
Layer 1: CharacterMesh0 (Driver — hidden)
  - AnimBP: SandboxCharacter_CMC_ABP (Motion Matching)
  - HiddenInGame=true, CastShadow=false
  - Root Motion → CMC
  - Mutable (Body_CSK) generates cosmetics
        |
        | Mesh synced at runtime (SetSkeletalMeshAsset)
        v
Layer 2a: WorldBodyMesh (visible to other players)
  - AnimBP: ABP_GenericRetarget (tag-based, universal standard)
  - ComponentTag[0] = "RTG_AutoGenerated"
  - bOwnerNoSee=true, bCastHiddenShadow=true
  - AlwaysTickPoseAndRefreshBones (CopyPose needs fresh bones)
        |
        | CopyPose from Mesh (FAnimNode_CopyPoseFromMesh)
        v
Layer 2b: LocalBodyMesh (visible only to owner)
  - AnimInstance: ULocalBodyAnimInstance (C++ Fast Path)
  - CopyPose(WorldBodyMesh) + ModifyBone(root) drift correction
  - bOnlyOwnerSee=true, CastShadow=false
```

### 2.2 Component Hierarchy

```
CapsuleComponent (Root)
  ├── CharacterMesh0 (Driver)     ← LAYER 1: hidden, runs MM
  │     ├── Body_CSK              ← Mutable body cosmetics
  │     ├── Head                  ← LeaderPose(Driver), bOwnerNoSee
  │     │     └── Head_CSK        ← Mutable head cosmetics
  │     ├── WorldBodyMesh         ← LAYER 2a: world body
  │     └── LocalBodyMesh         ← LAYER 2b: local body
  │           └── Local_Body_CSK  ← Mutable local body cosmetics
  └── FirstPersonCamera           ← On capsule, offset (23, 0, 62)
```

### 2.3 Visibility Matrix

| Component | HiddenInGame | OwnerNoSee | OnlyOwnerSee | CastHiddenShadow | CastShadow |
|-----------|:-:|:-:|:-:|:-:|:-:|
| Driver (GetMesh) | true | - | - | - | false |
| WorldBodyMesh | false | true | - | true | true |
| Head | - | true | - | true | true |
| Groom components | - | true | - | true | true |
| LocalBodyMesh | false | - | true | - | false |

**What the player sees:**
- Looking down → LocalBodyMesh (legs, arms, body with root correction)
- Shadow → WorldBodyMesh (CastHiddenShadow)
- Other players see → WorldBodyMesh + Head (full body)

### 2.4 Pose Transfer Methods

| Hop | Source → Target | Method | Why |
|-----|----------------|--------|-----|
| Driver → WorldBody | CharacterMesh0 → WorldBodyMesh | ABP_GenericRetarget | Universal retarget standard; config-only skeleton swaps |
| WorldBody → LocalBody | WorldBodyMesh → LocalBodyMesh | CopyPose from Mesh | Needs post-copy bone modification (ModifyBone) |
| Driver → Head | CharacterMesh0 → Head | LeaderPose | No modification needed; zero game-thread cost |

### 2.5 Animation Costs

| Component | Method | Cost |
|-----------|--------|------|
| Driver | Full MM + locomotion graph | High |
| WorldBodyMesh | ABP_GenericRetarget (identity retarget) | Low |
| Head | LeaderPose | Zero |
| LocalBodyMesh | CopyPose + ModifyBone(root) | Low |

Render: per viewer only one body variant visible (OwnerNoSee/OnlyOwnerSee culling).

## 3. SpineLock — Root Drift Compensation

### 3.1 How It Works

Camera is the authority. SpineLock measures how far neck_02 drifted from the camera and moves the root bone inversely to pull the body back under the camera.

**AnimGraph chain:**
```
CopyPoseFromMesh → LocalToCS → ModifyBone(root) → CSToLocal → Output
```

### 3.2 Algorithm (per frame)

1. **Game thread** (`NativeUpdateAnimation`): read root and neck_02 from **WorldBodyMesh** (not LocalBody — avoids feedback loop)
2. **Anim thread** (`PreUpdate`): convert camera position to mesh Component Space
3. Calculate drift: `Drift = neck_02_CS - Camera_CS`
4. Apply dead-zones (absorb micro-sway)
5. Move root: `RootPosition = WorldRoot - Drift`

### 3.3 Dead-Zones

| Axis | Threshold | Purpose |
|------|-----------|---------|
| XY | 8 cm | Prevents foot glide during idle breathing/sway |
| Z | 5 cm | Ignores small vertical oscillations; corrects crouch/jump gaps |

If drift < threshold → ignored. If drift > threshold → only the excess is corrected.

### 3.4 ModifyBone Configuration

| Parameter | Value | Purpose |
|-----------|-------|---------|
| Bone | `root` | Moves entire skeleton |
| TranslationMode | BMM_Replace | Full position override |
| TranslationSpace | BCS_ComponentSpace | Relative to mesh root |
| RotationMode | BMM_Ignore | Preserves breathing/lean |
| ScaleMode | BMM_Ignore | No scale changes |

### 3.5 Thread Safety

`FSpineLockData` bridges game thread → anim thread. Game thread writes bone positions, anim thread reads them in `PreUpdate`. One-frame lag on Z is acceptable.

### 3.6 Industry Assessment

**SpineLock is not a standard industry pattern.** It's a custom solution for ALIS's unique constraints (MM + full body + stable camera).

| # | Approach | Used By | Camera Stability | Full Body |
|---|----------|---------|:---:|:---:|
| 1 | Camera on bone | UE5 FPS Template, Insurgency | Low (jitter) | Yes |
| 2 | Separate FP arms | CoD, Battlefield 3/4 | Perfect | No |
| 3 | Root Motion Lock | Lyra, Fortnite | High | Yes |
| 4 | Unified FP/TP + IK | PUBG, Tarkov, Halo Infinite | Medium | Yes |
| 5 | **Drift compensation** | **ALIS** | **High** | **Yes** |

**Why ALIS can't use standard approaches:**
- Camera on bone → rejected (MM jitter, tested and rejected by user)
- Separate FP arms → rejected (need full body visible)
- Root Motion Lock → best long-term fix, but requires MM animation authoring, not code
- IK constraints → complements SpineLock, doesn't replace it

### 3.7 The Real Fix: Motion Matching Tuning

Root cause is MM animations with large root displacement during stops. If MM is tuned so overshoot < 5cm, SpineLock dead-zones absorb all drift — correction becomes invisible.

**MM tuning checklist (animation side):**

| Setting | Purpose |
|---------|---------|
| Distance Matching | Aligns foot plants to travel distance on stops |
| Stride Warping | Adjusts stride to capsule velocity |
| Pose Search cost weights | Penalize poses with large root offset |
| Transition animations | Dedicated stop/pivot anims returning root to origin |
| Root Lock in stop anims | Lock root during idle/stop blends |

**With proper MM tuning, SpineLock becomes a passive safety net.**

## 4. Visibility System

### 4.1 ApplyFirstPersonVisibility

Called in `BeginPlay` and on every `OnMutableMeshUpdated` callback. Mutable mesh regeneration resets visibility flags.

Sets:
- Driver → `HiddenInGame=true`, `CastShadow=false`
- WorldBodyMesh → `OwnerNoSee=true`, `CastHiddenShadow=true`
- LocalBodyMesh → `OnlyOwnerSee=true`
- Head/Groom → `OwnerNoSee=true`, `CastHiddenShadow=true`
- `HideBoneByName` on head/neck bones of LocalBodyMesh

### 4.2 Groom Visibility Fix

**Problem:** Groom components (hair, beard) visible in FP view.

**Root cause:** Groom components appear **asynchronously after Mutable update** — not present when `ApplyFirstPersonVisibility()` first runs.

**Fix:** Deferred retry via `FTimerHandle` — re-apply visibility 0.5s after `OnMutableMeshUpdated`. Catches `GroomComponent_0`, `GroomComponent_1`, and associated `NiagaraComponent` (hair physics).

**Filter criteria:** Component name or class name contains: `Head`, `Groom`, `Hair`, `Beard`, `Eyebrow`, `Eyelash`, `Mustache`.

### 4.3 AlwaysTickPoseAndRefreshBones

Required on WorldBodyMesh and LocalBodyMesh. Without it, UE stops updating bones when mesh is hidden via `OwnerNoSee` — breaks CopyPose and HideBoneByName.

## 6. Camera System

### 6.1 Current State

- Camera on capsule at `(23, 0, 62)` — chin/nose level
- `bUsePawnControlRotation = true`
- FOV = 90° (standard FPS)
- No spring arm

### 6.2 Crouch Camera — NOT IMPLEMENTED

Camera stays at Z=62 during crouch. `BaseEyeHeight` doesn't work with `UCameraComponent` on capsule (only works with `PlayerCameraManager` default pipeline).

**UE defaults:**
| Property | Value |
|----------|-------|
| BaseEyeHeight | 64.0 (standing) |
| CrouchedEyeHeight | 40.0 |
| Capsule half-height standing | 88.0 |
| Capsule half-height crouched | 40.0 |

**Recommended approach:** Override `OnStartCrouch`/`OnEndCrouch`, lerp camera Z with `FInterpTo`:
```
Standing:  Z = 62
Crouched:  Z ≈ 34 (proportional to capsule shrink)
Speed:     5.0 (~0.3s transition)
```

### 6.3 Camera Height Alternatives

| Approach | Pros | Cons | For |
|----------|------|------|-----|
| OnStartCrouch/OnEndCrouch | Event-driven, no Tick | Needs FInterpTo for smooth transition | Crouch drop |
| UCameraModifier | Decoupled, reusable | Needs PlayerController | Pitch-based Z |
| BaseEyeHeight | Built-in | Doesn't work with UCameraComponent | Standard capsule cameras |
| Bone-attached camera | Zero code | Jitter (rejected) | Animation-driven FP |

## 10. Experiment Log

### 10.1 SpineLock Iterations

| # | Approach | Result |
|---|----------|--------|
| 1 | ModifyBone(spine_05) XY Lock | ❌ Lower spine slides, visually unchanged |
| 2 | FABRIK IK (spine_01→spine_05) | ❌ Extreme mesh tearing — overshoot exceeds spine length |
| 3 | **Camera-Centric Root Offset** | ✅ Entire mesh slides to neutralize drift, no tearing |

### 10.2 Camera Framing Trials

| # | Offset | Result |
|---|--------|--------|
| 1 | X=12, Z=150 | ❌ Too forward, neck clips |
| 2 | X=-10, Z=155 | ⚠️ Partial, limited look-down |
| 3 | X=-25, Z=165 | ❌ Too far back |
| 4 | X=2, Z=152 | ✅ Golden mean (Mar 26) |
| 5 | X=23, Z=62 | ✅ Current (Mar 28) — chin/nose level with capsule-relative offset |

### 10.3 Anchor Bone Experiments (Mar 26 Morning)

| # | Method | Result | Why Failed |
|---|--------|--------|-----------|
| 1 | Pelvis as anchor | ❌ Off-center | Locking pelvis moves entire skeleton including legs |
| 2 | Capsule-relative XY per frame | ❌ Drift/jitter | CS origin ≠ capsule center; chases animated root |
| 3 | Camera-relative XY projection | ❌ Off-center | Camera offset doesn't map 1:1 to bone CS |
| 4 | Pelvis ZeroRotator | ❌ Body on its side | Pelvis ref rotation is (87.9, 90, 90), not zero |
| 5 | Pelvis ref pose rotation | ❌ Can't turn | Absolute rotation freeze prevents facing changes |
| 6 | spine_03 pitch BMM_Replace | ❌ Contorted | Replace destroys bone rest rotation |
| 7 | spine_03 pitch BMM_Additive | ❌ Extreme lean | Full camera pitch on one bone too aggressive |
| 8 | 5-bone distributed pitch | ❌ Forward lean | BP bUseControllerRotationPitch=true compounded tilt |
| 9 | Negated camera pitch | ❌ Opposite lean | Root cause was BP settings, not pitch sign |
| 10 | BP_Hero audit | Root cause found | bUseControllerRotationPitch=true in BP |

**Key takeaway:** Always audit BP overrides before C++ experiments.

### 10.4 Initialize Lifecycle

`FAnimInstanceProxy::Initialize` runs 4 times (UE double-init + MM startup + Mutable RecalcRequiredBones). Fix: persistent state lives in `ULocalBodyAnimInstance`, not proxy.

## 11. Hand-Leg Penetration Analysis

**Symptom:** Hands penetrate legs during idle/walk. Visible on WorldBodyMesh too — not caused by SpineLock.

**Root cause:** Source MM animations + Mutable clothing volume (jeans thicker than mannequin).

**Fix:** Post-process hand IK in AnimGraph (detect hand-thigh proximity, push outward). Priority: Low — cosmetic.
