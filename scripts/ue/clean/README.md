# Clean Scripts

PowerShell scripts to clean project and plugin build artifacts (Binaries/Intermediate directories).

**Opt-in clean:** Use `--clean` flag when you need clean rebuild. Default is incremental build (faster, uses UE's smart build system).

---

## Quick Reference

```powershell
# Clean all (plugins + project)
.\clean_all.ps1

# Clean plugins only
.\clean_plugins.ps1

# Clean project only
.\clean_project.ps1

# Clean specific plugin
.\clean_plugins.ps1 -PluginName ProjectLoading

# Verbose output (show sizes)
.\clean_all.ps1 -Verbose
```

---

## When to Clean (Important!)

**UE Build System (UnrealBuildTool) supports incremental builds** - it only rebuilds what changed.

**Clean is opt-in** because forcing clean every time:
- ❌ Defeats UE's smart incremental build system
- ❌ Wastes time (forces full rebuild)
- ❌ Makes development iteration painfully slow

**When you SHOULD clean:**
- Switching git branches with major changes
- Build artifacts corrupted (linker errors, weird behavior)
- After major refactoring
- CI/CD final validation (optional)
- Switching UE engine versions

**When you should NOT clean (most of the time):**
- Regular development iteration
- Small code changes
- Bug fixes
- Testing after minor edits

**Default: NO clean** - trust UE's incremental build!

## Scripts

### clean_all.ps1
Clean all project and plugin artifacts (most common).

**Usage:**
```powershell
.\clean_all.ps1 [-Verbose]
```

**What it does:**
- Calls `clean_plugins.ps1` (all plugins)
- Calls `clean_project.ps1`
- Shows summary of cleaned artifacts

**Use when:**
- Before full rebuild
- After switching branches
- When build artifacts are corrupted

---

### clean_plugins.ps1
Clean plugin Binaries/Intermediate directories.

**Usage:**
```powershell
# Clean all plugins
.\clean_plugins.ps1

# Clean specific plugin
.\clean_plugins.ps1 -PluginName ProjectLoading

# Verbose output
.\clean_plugins.ps1 -Verbose
```

**What it does:**
- Scans `Plugins/Boot/`, `Plugins/Core/`, `Plugins/Foundation/`, `Plugins/Systems/`, `Plugins/UI/`, `Plugins/Features/`
- Removes `Binaries/` and `Intermediate/` from each plugin
- Shows count of cleaned directories

**Use when:**
- Single plugin needs rebuild
- Plugin compilation errors
- Hot reload issues

---

### clean_project.ps1
Clean project-level Binaries/Intermediate directories.

**Usage:**
```powershell
.\clean_project.ps1 [-Verbose]
```

**What it does:**
- Removes `Binaries/` from project root
- Removes `Intermediate/` from project root
- Does NOT touch `DerivedDataCache/` (uncomment to enable)

**Use when:**
- Main game module needs rebuild
- Linker errors
- Project-level compilation issues

## Integration with Build/Test Scripts

### test.py (Default: Incremental Build)
```bash
# Incremental build + test (default, FAST - 30 sec boot validation)
python scripts/ue/standalone/test.py

# Clean + full rebuild + test (slow, use when needed)
python scripts/ue/standalone/test.py --clean

# Skip build entirely (use existing binaries - 30 sec boot validation)
python scripts/ue/standalone/test.py --no-build
```

### build_standalone.bat (Default: Incremental Build)
```cmd
REM Incremental build (default, FAST)
scripts\ue\standalone\build_standalone.bat

REM Clean + full rebuild (slow, use when needed)
scripts\ue\standalone\build_standalone.bat /clean
```

## SOLID Structure

These scripts follow SOLID principles:

**Single Responsibility:**
- `clean_plugins.ps1` - Cleans plugins ONLY
- `clean_project.ps1` - Cleans project ONLY
- `clean_all.ps1` - Orchestrates both

**Reusability:**
- Called by test.py, build_standalone.bat, manual use
- Specific plugin cleaning via `-PluginName` parameter

**Separation:**
- Clean operations in `scripts/ue/clean/`
- Build operations in `scripts/ue/build/`
- Test operations in `scripts/ue/test/`

## What Gets Cleaned

### Plugin Artifacts
```
Plugins/
  Boot/
    Orchestrator/
      Binaries/      <- REMOVED
      Intermediate/  <- REMOVED
  Core/
    ProjectCore/
      Binaries/      <- REMOVED
      Intermediate/  <- REMOVED
  Systems/
    ProjectLoading/
      Binaries/      <- REMOVED
      Intermediate/  <- REMOVED
```

### Project Artifacts
```
Alis/
  Binaries/          <- REMOVED
  Intermediate/      <- REMOVED
  DerivedDataCache/  <- NOT removed (optional)
```

**Safe to clean:** Yes. All artifacts are regenerated by UE build system.

## Performance

**Clean time:**
- All plugins: 2-5 seconds
- Project only: 1-2 seconds
- Total: 3-7 seconds

**Disk space freed:**
- Typical: 500 MB - 2 GB (depends on build artifacts)
- Use `-Verbose` flag to see exact sizes

## Troubleshooting

### Error: "Access Denied"
**Cause:** UE Editor or UE processes holding files

**Fix:**
```powershell
# Kill UE processes first
taskkill /F /IM UnrealEditor.exe
taskkill /F /IM UnrealEditor-Cmd.exe

# Then clean
.\clean_all.ps1
```

### Error: "Cannot remove item"
**Cause:** Files in use by antivirus or other process

**Fix:**
- Close antivirus real-time scanning temporarily
- Close Visual Studio
- Retry clean

### Warning: "Plugin clean failed"
**Cause:** Some files couldn't be removed (non-critical)

**Effect:** Build will continue with existing artifacts

**Fix (if needed):**
- Close all UE/VS instances
- Manually delete problematic directories
- Retry clean

## Manual Clean (Alternative)

If PowerShell scripts fail, manual clean:

```cmd
REM Navigate to project root
cd <project-root>

REM Remove project artifacts
rmdir /s /q Binaries
rmdir /s /q Intermediate

REM Remove plugin artifacts (example)
rmdir /s /q Plugins\Systems\ProjectLoading\Binaries
rmdir /s /q Plugins\Systems\ProjectLoading\Intermediate
```
